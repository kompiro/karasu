# AT-0007: QA チェックリスト生成スキル

- **日付**: 2026-03-24
- **関連ADR**: ADR-20260324-01
- **対象**: `.claude/skills/qa/SKILL.md`

## 概要

`/qa` スキルが `docs/acceptance/*.md` を読み込み、自動検証を実行しつつ手動確認項目をまとめたチェックリストを生成することを検証する。

## 受け入れ条件

### AC-1: チェックリストの生成

- [ ] `/qa` を実行すると `docs/qa/YYYY-MM-DD-checklist.md` が生成される（今日の日付）
- [ ] `docs/qa/` ディレクトリが存在しない場合でも自動的に作成される
- [ ] 生成後にファイルパスとサマリー（自動検証結果・手動確認件数）が出力される

### AC-2: 自動検証コマンドの実行

- [ ] AT ファイルに含まれる bash コードブロックのコマンドが実行される
- [ ] 実行結果（成功 `[x]` / 失敗 `[ ]`）がチェックリストに記録される
- [ ] 同一コマンドが複数 AT に存在する場合、1回だけ実行される
- [ ] 参照元の AT 番号がコマンド行に記載される（例: `(AT-0001, AT-0006)`）

### AC-3: 手動確認項目の収集

- [ ] AT ファイルの `- [ ]` 項目がすべてチェックリストに含まれる
- [ ] `- [x]` 済みの項目は含まれない
- [ ] AT ファイルごとにグループ化され、AT 番号とタイトルが見出しとして表示される
- [ ] 見出しが元の AT ファイルへのリンクになっている（例: `[AT-0006: ...](docs/acceptance/0006-...md)`）
- [ ] AC のグループ構造（`AC-N`, `AC-N.M`）が維持される

### AC-4: 手動確認項目がない場合

- [ ] すべての AT が自動検証のみの場合、「手動確認項目なし」と明記される
