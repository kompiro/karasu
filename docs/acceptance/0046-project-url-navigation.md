# AT-0046: Project URL Navigation

## 概要

ProjectMode でアクティブなプロジェクトが URL パス（`/projects/<uuid>`）に反映され、
ブラウザの戻る/進むボタンやリロード・URL 共有でプロジェクトが復元されること。

## 前提条件

- ProjectMode（OPFS モード）で起動していること
- 複数のプロジェクトが存在すること

## テストケース

### TC-1: 初回起動時に URL が正規化される

1. URL に `/projects/` パスがない状態でアプリを開く
2. プロジェクト一覧が読み込まれる
3. URL が `/projects/<最初のプロジェクト ID>` に自動更新される

### TC-2: URL でプロジェクトを直接指定して開く

1. `/projects/<プロジェクト B の ID>` の URL をブラウザに入力して開く
2. プロジェクト B が自動的に選択された状態で表示される

### TC-3: プロジェクト切り替えで URL が変わる

1. プロジェクト A を表示している状態でプロジェクト B に切り替える
2. URL が `/projects/<プロジェクト B の ID>` に変わる

### TC-4: ブラウザの戻るボタンでプロジェクトが戻る

1. プロジェクト A → プロジェクト B の順に切り替える
2. ブラウザの戻るボタンを押す
3. プロジェクト A に戻り、URL も `/projects/<プロジェクト A の ID>` になる

### TC-5: ブラウザの進むボタンでプロジェクトが進む

1. TC-4 の状態でブラウザの進むボタンを押す
2. プロジェクト B に戻り、URL も `/projects/<プロジェクト B の ID>` になる

### TC-6: URL をコピーしてリロードしてもプロジェクトが復元される

1. プロジェクト B を表示している状態で URL をコピーする
2. 新しいタブでその URL を開く（または現在のページをリロードする）
3. プロジェクト B が表示される

### TC-7: プロジェクト切り替え時にドリルダウン状態はリセットされる

1. プロジェクト A でドリルダウン（`#krs-system-Payment` など）した状態でプロジェクト B に切り替える
2. URL が `/projects/<プロジェクト B の ID>`（hash なし）になる
3. プロジェクト B のルートビューが表示される

### TC-8: URL とハッシュの組み合わせで復元される

1. `/projects/<プロジェクト A の ID>#krs-system-Payment` の URL を開く
2. プロジェクト A が選択され、`Payment` ノードのドリルダウンビューが表示される

## 実装ファイル

```bash
packages/app/src/hooks/useProjectNavigation.ts
packages/app/src/ProjectModeApp.tsx
```
