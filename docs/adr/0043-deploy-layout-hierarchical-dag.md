# ADR-0043: Deploy 図レイアウト — 階層 DAG レイアウト（Longest Path Layering）

- **日付**: 2026-04-08
- **ステータス**: 決定済み
- **関連**: Issue #392, [deployment-diagram (ADR-0011)](0011-deployment-diagram-design.md)

## 背景

deploy 図のレイアウト（`deploy-layout.ts`）はコンテナを一律に左から右へ並べていた。`examples/deploy/system.krs` のように 7 つのコンテナがある場合、図の幅が 2000px を超え、プレビュー UI・エクスポート SVG ともに非常に見づらくなっていた。

## 決定

階層 DAG レイアウト（Sugiyama 風の Longest Path Layering）を採用し、`DeployViewSlice.ghostEdges` が表すサービス間依存グラフをもとに、上流コンテナを上、下流コンテナを下に配置する「レイヤー分け + 横並べ」方式に変更する。

### アルゴリズム

1. **レイヤー割り当て**: in-degree = 0 のノードからの BFS による Longest Path Layering
2. **同レイヤー水平配置**: 初期実装は挿入順、左から右へ
3. **Y 座標**: `rowY[i] = rowY[i-1] + maxHeightOfLayer[i-1] + ROW_GAP`（`ROW_GAP = 64`、`CONTAINER_GAP` より大きく）
4. **Ghost エッジルーティング**: 同レイヤーは左右接続、隣接/長距離レイヤーは下辺中央 → 上辺中央
5. **未分類コンテナ**（`realizes` なし）: 最下段の独立行に配置
6. **サイクル**: BFS 中に検出した後退辺はレイヤー割り当てに使わず ghost エッジとして描画

## 理由

- deploy 図の主目的は「どのサービスがどこで動いているか・関係サービスは何か」を伝えることであり、依存関係を視覚的に反映したレイアウトが目的に適合する
- 既存の `LayoutResult` 型・SVG レンダラーを変更せずに済む（座標値の変化のみ）
- 横展開の問題を根本的に解消できる
- 未分類コンテナを最下段に分離することで、「依存グラフに属さない」ことが意味的に正確に表現される
- ドリルダウンやハイライト連携など将来拡張との親和性が高い

## 却下した案

### 案1: Wrap レイアウト（行折り返し）

`MAX_DIAGRAM_WIDTH` 超過時に折り返す案。実装は単純だが、コンテナの意味的な関係（service の依存関係）を反映しない。

### 案2: 固定グリッドレイアウト（N×M 均等分割）

`√N` 近似で列数を決める案。コンテナ幅がばらつく場合に均等割が機能せず、依存関係も反映しない。
