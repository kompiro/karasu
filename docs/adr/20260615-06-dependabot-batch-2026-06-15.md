# ADR-20260615-06: Dependabot Batch Triage (2026-06-15) — `actions/checkout` 6.0.3

- **日付**: 2026-06-15（月曜）
- **ステータス**: 決定済み
- **関連**:
  - PR [#1607](https://github.com/kompiro/karasu/pull/1607) — `chore(deps): bump actions/checkout from 6.0.2 to 6.0.3`
  - [ADR-20260329-01](./20260329-01-dependabot.md) — Dependabot 採用
  - [ADR-20260519-05](./20260519-05-dependabot-batch-2026-05-19.md) — 前回のバッチ triage

## 背景

2026-06-15 時点で開いていた Dependabot version update PR 1 本を triage した。サプライチェーン攻撃のリスクを踏まえ、bump 種別を問わず upstream まで遡ってリスク分析した。

| PR | 対象 | bump | 変更概要 | CI |
|----|------|------|----------|----|
| [#1607](https://github.com/kompiro/karasu/pull/1607) | `actions/checkout` 6.0.2 → 6.0.3 | patch | SHA-256 リポジトリの checkout 初期化修正と merge-commit SHA 正規表現の拡張のみ | 全 check green（secret-gated job は skip）|

### リスク分析の結論

- **リリースノート / CHANGELOG**: v6.0.3 は SHA-256 リポジトリでの checkout 初期化修正（[actions/checkout#2439](https://github.com/actions/checkout/pull/2439)）と merge-commit SHA 正規表現の拡張＋テスト追加（[actions/checkout#2414](https://github.com/actions/checkout/pull/2414)）の 2 点。機能修正のみ。
- **コード差分**: `de0fac2…df4cb1c` の compare を確認。変更は `src/`（`git-command-manager.ts` / `git-source-provider.ts` / `github-api-helper.ts` / `input-helper.ts` / `ref-helper.ts`）＋テスト＋コンパイル済み `dist/index.js` のみ。内容は `git init --object-format=sha256` 対応と `GET /repos/{owner}/{repo}/hash-algorithm` での object format 判定の追加で、リリースノートと完全に整合。不審なネットワーク送信・難読化・exfiltration の類はなし。
- **メンテナ・所有権の変化**: publisher は `actions` org のまま。コミットは `ericsciple`（既存コアメンテナ）と新規 contributor `yaananth`（actions org の PR レビュー経由）。リポジトリ移管・改名なし。
- **lifecycle スクリプト**: GitHub Action のため postinstall / prepare 等なし。追加なし。
- **依存ツリー**: 新規 transitive 依存の追加なし（src/dist の変更のみ）。
- **既知 advisory**: 該当なし。
- **pin 方式**: 本リポジトリは全 9 ワークフローで `actions/checkout` を**フル commit SHA pin**（`# v6.0.2` コメント付き）。PR は SHA を `df4cb1c…` に更新し、良好なサプライチェーン慣行を維持する。

リスクレベル **low**。

## 決定

- **PR #1607 を squash merge する**（マージ済み）。`actions/checkout` のパッチ更新で CI 専用、低リスク。
- `.github/dependabot.yml` は変更しない。

## 理由

- CI 専用の GitHub Action で、ランタイム成果物（core / app / cli / lsp / vscode）には影響しない。
- 全 required check が green で、secret-gated job（Playwright / ExTester / Cloudflare）は bot PR ポリシー通り skip。
- upstream まで遡った確認で配布主体・コード・lifecycle スクリプトに不審点がなく、変更内容はリリースノートと整合する。
- 全ワークフローで SHA pin を維持しており、タグ改ざんによる差し替えリスクを受けない。

## 却下した案

- **マージを見送り cooldown を待つ案**: 採用しなかった。本リポジトリの cooldown（全 semver レベル 7 日）は npm 依存のサプライチェーン対策が主眼で、フル SHA pin される GitHub Action のパッチ更新かつ差分がリリースノートと整合する本件では待機の積極的理由がない。

## スコープ外

- `actions/checkout` の major / minor 更新（本 triage 時点では未起票）。
- 次回以降のバッチで複数 PR の競合や CI 失敗が発生する場合は、本 ADR と同様にバッチ ADR で triage する。
