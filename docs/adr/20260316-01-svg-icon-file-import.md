# ADR-20260316-01: SVGアイコンファイルの外部インポート方式

- **日付**: 2026-03-16
- **ステータス**: 決定済み

## 背景

karasu のレンダラーにシェイプレジストリ（`shape-registry.ts`）を導入し、`registerIcon()` で外部シェイプを登録できるようになった。しかし SVG アイコンの定義方法として、SVG ファイルを直接扱える仕組みが必要だった。

svg-icon スキルで作成したアイコンをそのまま `.svg` ファイルとして保持し、レンダラーに取り込む構造が求められた。

## 決定

### SVGファイル直接保持 + マニフェスト方式

1. **SVG ファイルは `.svg` のまま保持する**（TS リテラルに埋め込まない）
2. **`icons.json` マニフェスト**でアイコン名とファイルパスを定義する
3. **テキストスロット規約**: SVG 内に `class="krs-label"` / `class="krs-description"` を持つ `<text>` 要素を配置し、ローダーが位置情報を抽出してレンダラーに渡す
4. ファイル読み込みは利用側（app など）の責任とし、core は Pure TS を維持する

### 却下した案: TS 文字列リテラルへの埋め込み

SVG 内容を TypeScript ファイル内の文字列リテラルとして管理する方式。
SVG ファイルとしてプレビューできなくなるため却下した。アイコンの視認性確認やデザイン調整には `.svg` ファイルを直接開けることが重要である。

## 理由

- **プレビュー可能性**: `.svg` ファイルをエディタやブラウザで直接プレビューできる
- **テキスト配置の明示性**: サンプルテキスト付きでプレビューすることで、完成形のレイアウトが SVG ファイル単体で確認できる
- **関心の分離**: core パッケージはファイル I/O を持たず、SVG 文字列の解析と登録のみを担当する
- **拡張性**: マニフェスト形式により、アイコンの追加・削除が JSON 編集のみで完結する
