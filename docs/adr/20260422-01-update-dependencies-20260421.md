# ADR-20260422-01: Dependency Updates — 2026-04-21

## Status

Accepted

## Context

First Dependabot batch after #776 consolidated the npm watcher to root-only and added a 7-day cooldown window. Two PRs were opened on 2026-04-21:

| Package | From | To | Type | PR | Status |
|---------|------|----|------|----|--------|
| `oxfmt` | 0.44.0 | 0.45.0 | dev (root) | #781 | Check ✓ |
| `marked` | 17.0.6 → 18.0.0 (vscode) | 18.0.0 | runtime (vscode, **and app side-effect**) | #783 | Check ✓ but regression |

The Cloudflare Pages deploy failure on Dependabot PRs remains expected (no secrets on fork PRs).

### oxfmt 0.44.0 → 0.45.0 — adopt

Release notes (from the `apps_v1.60.0` tag in the `oxc-project/oxc` monorepo):

- Upstream Prettier sync bumped to 3.8.2 — formatting output may shift in line with Prettier 3.8.x behavior.
- Formatting fixes: non-idempotent TS comment formatting; dangling comments in empty enums indent with block indent; extra outer parens on `return` with JSDoc type casts removed; trailing comma preserved in TSX arrow functions with default type params; newline preserved between self-closing JSX and single-char text; Unicode char escaping.
- `.editorconfig` `quote_type` support added — existing `.editorconfig` could now influence quote style where it previously did not (we have no `.editorconfig` in the repo, so no impact).
- CLI: `vite.config.ts` recognition now gated behind `VP_VERSION` env var; new hint when all files are ignored; consecutive leading slashes in paths handled; writer flush errors surfaced.

Dev-tooling only. `Check` is green. Running `pnpm run format:check` after the upgrade would flag any source-file drift from the Prettier 3.8.2 sync; if clean, merge as-is.

### marked 17.0.6 → 18.0.0 (#783) — **do not adopt**

#783 was opened because `packages/vscode/package.json` was still pinned at `^17.0.6` after the 2026-04-20 batch (#778 only bumped `packages/app`). Dependabot's root-scope resolution now wants to unify all workspace uses of `marked`, which is the right direction. But the generated PR has two problems:

1. **It downgrades `packages/app` from `^18.0.2` (merged via #778 on 2026-04-21) to `^18.0.0`.** This is a regression — we lose the 18.0.1 minifier-safe lookbehind regex fix and the 18.0.2 infinite-loop fix for blank-line-inside-indented-code-block.
2. **It targets `18.0.0`, not the latest `18.0.2`.** Likely because `18.0.2` was published within the new 7-day cooldown window and is being held back, while `18.0.0` was released long enough ago to be eligible.

Decision: close #783 without merging. Leave `packages/vscode` on `^17.0.6` for now. Once `18.0.2` clears the cooldown window (a few days from now), Dependabot will re-open a PR targeting `18.0.2`, which will unify the workspace cleanly without regressing `packages/app`. No manual replacement PR is warranted at this point — the downside of waiting is only that `packages/vscode` stays on `marked` 17 for a few more days, and it does not exercise the v18 correctness fixes that motivated the app upgrade (vscode extension markdown rendering is not a hot path for the issues 18 fixes).

## Decision

| PR | Package | Outcome |
|----|---------|---------|
| #781 | oxfmt 0.44.0 → 0.45.0 | Merged (CI `Check` green, `format:check` clean) |
| #783 | marked 17.0.6 → 18.0.0 (vscode) | Closed without merging — would have regressed `packages/app` 18.0.2 → 18.0.0. Wait for Dependabot to re-propose against 18.0.2 once the cooldown window expires. |

## Consequences

**Positive:**

- `oxfmt` tracks the Prettier 3.8.x behavior line, reducing future churn.
- The 7-day cooldown policy added in #776 is doing exactly what it was set up to do — holding back a freshly published release and forcing us to reason about whether we want the older version. We keep the policy as-is so future updates stay subject to the same gate.

**Negative:**

- `packages/vscode` stays on `marked` 17 for the rest of the cooldown window. Low-impact: the extension's markdown rendering is not a hot path for the issues v18 fixes, and the split will self-resolve on the next Dependabot run.
