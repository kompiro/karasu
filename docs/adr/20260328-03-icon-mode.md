# ADR-0041: アイコンモード — SVG アイコンによるノード表示切り替え

- **日付**: 2026-03-28
- **ステータス**: 決定済み
- **関連**: Issue #30, Issue #81, [ADR-0042](20260407-02-resource-shape-and-infra-icon-mode.md)

## 背景

karasu の既存シェイプ語彙（box, cylinder, user, cloud 等）は UML/DFD 時代の「紙＋モノクロ印刷」制約下で生まれたものであり、色とアイコンを活用できる現代の SVG 表示環境では必ずしも最適ではない。特に `user` シェイプは TypeScript で比率計算を含む動的な実装になっており、見た目を調整するためにコードを変更しなければならなかった。アイコンで要素を識別できるなら、シェイプの多様性よりも視認性と調整のしやすさを優先する設計が望ましい。

## 決定

UI 層でスタイルシートを差し替える方式（案 C）を採用する。`packages/core/src/builtins/icon-theme.ts` に `ICON_THEME_STYLE_SOURCE` を定義し、アプリ側が `displayMode === "icon"` のときに `styleSource` の前に注入する。

```typescript
const effectiveStyle = displayMode === "icon"
  ? ICON_THEME_STYLE_SOURCE + "\n" + userStyleSource
  : userStyleSource;
```

### カードレイアウト（固定サイズ2種）

- **説明あり**: 160×100（黄金比近似 1.60 ≈ φ）
- **説明なし**: 160×56（圧縮版）

各ノードは「タイトル行（28px）」「説明行（44px）」「アクションバー（28px）」で構成し、アクションバー（🔗 リンク数 / 👥 チーム / [ⓘ] 詳細パネル）は React HTML オーバーレイで描画する。

### コンテナノードのヘッダー

`system` / `service`（子あり）/ `domain` はカード型ではなく「ヘッダーバンド + 子ノード領域」で描画する。SVG ファイル内の `<g class="krs-pictogram">` を leaf カードとコンテナヘッダー両方で再利用する。

### モード切替

モード切替時は全体を再レイアウトする。シェイプモードの可変サイズとアイコンモードの固定サイズは共存しない。

## 理由

- **UI 層でのスタイル注入**: `core` の API（`compile()`）を変えずに済み、SVG Export 時も同じ `ICON_THEME_STYLE_SOURCE` を使えば一貫した描画が得られる。ユーザーは `.krs.style` でさらに上書きできる
- **固定サイズカード**: アイコンによる識別が可能になる前提なら、ノードサイズを動的計算する必要がなく、レイアウトが安定する
- **アクションバーを React HTML で描画**: SVG Export 時は静的表示（HTML オーバーレイなし）、アプリ内ではインタラクティブになり、両ユースケースが両立する
- **クリックゾーン分離**: 「タイトル/説明領域 = ドリルダウン」「[ⓘ] = 詳細パネル」と物理的に分離されるため、`data-has-children` による条件分岐が不要になる
- **ピクトグラム再利用**: 1 SVG ファイルで leaf カードとコンテナヘッダーの両用途に対応できる

## 却下した案

### 案A: `.krs` ファイル内ディレクティブ（`@display-mode: icon`）

表示の関心事がドキュメントに入り込む。Export 用途と閲覧用途で別設定が持てない。

### 案B: コンパイル API パラメータ（`compile(src, style, path, { theme: "icon" })`）

呼び出し側が制御できる利点はあるが、core の API に表示テーマの概念が入り込む。

## 残課題

- 20px ピクトグラムの視認性 — 単純なシルエット形状（塗り潰し系）が向く
- AI サマライズによる文字数超過対応は将来課題（別 Issue）
