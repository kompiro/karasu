# ADR-0006: YAML スタイル構文移行の見送り

- **日付**: 2026-03-23
- **ステータス**: 決定済み
- **関連**: [GitHub Issue #7](https://github.com/kompiro/karasu/issues/7), [GitHub Issue #13](https://github.com/kompiro/karasu/issues/13), [GitHub Issue #18](https://github.com/kompiro/karasu/issues/18)

## 背景

深いネスト（4〜5段）でブレース構文の閉じブレースが連続し読みにくいという問題から、
`:` + インデントによる YAML スタイルへの移行を検討した。

検討した案は3つ。

| 案 | 概要 |
|----|------|
| 案1: 完全移行 | `{ }` を廃止し `:` + インデントのみにする |
| 案2: ハイブリッド | `{ }` と `:` + インデントの両方を許容する |
| 案3: プロパティのみ | ブロックはブレース維持、プロパティ値のみ `:` 記法に変更する |

## 決定

**YAML スタイルへの移行を行わない。ブレース構文（`{ }`）を継続する。**

## 理由

1. **エッジ宣言との根本的な非整合**
   `Customer -> ECommerce "購入する"` はkarasuの中心的な記法だが、YAML 構造上に自然な置き場がない。
   `edges:` ブロックへの押し込みは可読性を損なう。

2. **「YAML ライクだが YAML ではない」問題**
   エッジ宣言が混在する時点で YAML の規則を破るため、YAML に慣れたユーザーを混乱させる。

3. **パイプ記法の実装コスト**
   `description: |` の実現にはインデント検知（INDENT/DEDENT トークン）が必要であり、
   案3（プロパティのみ変更）は「最小コスト」という前提が崩れる。

4. **現行ブレース構文で十分**
   `description` の複数行は `"""` トリプルクォートで解決済み。
   ブレース構文を維持することが最もシンプルな判断。
