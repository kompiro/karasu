# ADR-20260428-02: 依存更新バッチ — 2026-04-28

- **ステータス**: 決定済み
- **関連**:

## 背景

2026-04-28 の Dependabot 月曜バッチで以下 5 件が起票された。

| PR | パッケージ | 変更 | 種別 | リリース日時 (UTC) | PR 時点経過 | 結果 |
|----|------------|------|------|--------------------|-------------|------|
| #899 | `actions/github-script` | SHA `d746ffe` → `3a2844b` | github-actions | 2026-04-09 22:39 | 18.1 日 | merged |
| #900 | `oxfmt` | 0.45.0 → 0.46.0 | npm devDep | 2026-04-20 | 8 日 | merged |
| #901 | `oxlint` | 1.60.0 → 1.61.0 | npm devDep | 2026-04-20 | 8 日 | merged |
| #898 | `pnpm/action-setup` | 6.0.1 → 6.0.3 | github-actions | **2026-04-21 18:29** | **6.26 日** | merged（人間レビュー済み） |
| #902 | `marked` | 17.0.6 → 18.0.2 (vscode) | npm major | 2026-04-18 | 10.1 日 | close → 再提出 #908 |

加えて、本バッチをきっかけに 2 件の付随変更を行った。

- **#904 / #903**: Cloudflare Pages preview job を bot 作者の PR では skip する設定。Dependabot PR 全件で `Deploy Preview to Cloudflare Pages` が secret 不足で失敗していたノイズを解消。
- **ADR-20260428-01（PR #905 でマージ）**: 上記 skip 方針の一般化（`github.event.pull_request.user.type != 'Bot'` を secret 必須ジョブの `if:` に入れる）を ADR 化。

## 決定

- npm 4 件（#899/#900/#901/#902）はレビューの上で順次マージする。`#902 marked v18` のみ vscode 拡張のメジャー更新かつ Cloudflare Preview で動作確認したいので、**Dependabot PR を close し、人間作者の #908 として再提出**する（bot PR では preview job が skip されるため）。
- `#898 pnpm/action-setup 6.0.3` は cooldown 7 日に対し **6.26 日**で起票されており設定通りに動いていない疑い。今回は内蔵 pnpm が `packageManager: pnpm@10.33.0` で上書きされ実害ゼロのためマージするが、**Issue #909 として観測ログを残し、来週以降再発するか追跡する**。

## 理由

- **メジャー更新（marked v18）の検証導線を確保するため、bot PR を close して人間 PR で出し直す。** ADR-20260428-01 の方針上、bot PR では Cloudflare Preview が常に skip される。`packages/app` は既に v18 系のため preview に直接の差分は出ないが、ロックファイル解決とビルドが壊れていないことを preview で観測するのが今回の major bump で最も安価な検証手段。
- **cooldown 違反は Dependabot 側の挙動疑義として扱い、設定の書き換えに直行しない。** `.github/dependabot.yml` は `default-days: 7` を明示しており、設定誤りではない。`github-actions` ecosystem 側で `cooldown` フィールドのサポートが限定的か、release time の取り方に揺れがある可能性が高いので、まず観測を継続して再現を確認してから upstream へ報告する。
- **devDep / minor / patch（oxfmt, oxlint, github-script）は通常通り順次マージする。** いずれも 7 日 cooldown を満たしており、CI の `Check` も SUCCESS のため追加検証は不要。

## 確認事項（来週以降）

- Issue #909 の観察項目: 次回月曜バッチ（2026-05-04）以降に `github-actions` ecosystem で 7 日未満のリリースが起票されないか。
- 再発した場合は `github/dependabot-core` に upstream で報告する。安全側に倒すなら `github-actions` の `default-days` を 10 まで引き上げる手もあるが、SHA pin 更新の遅延と引き換えなので Issue で議論してから決める。

## 却下した案

### Dependabot 設定をいま `default-days: 10` などに引き上げる

1 件の観測だけで設定を変えると、根本原因が分からないまま「動いているフリ」を作ってしまう。先に再現性を確認する方が手戻りが少ない。

### marked v18 を Dependabot PR のままマージする

bot PR では Cloudflare Preview が skip されるので、メジャー更新の preview 検証ができない。close → 人間 PR で再提出した方が ADR-20260428-01 の運用と整合的。
