# ADR-20260312-04: CSSインスパイアのスタイリングシステム

- **日付**: 2026-03-12
- **ステータス**: 決定済み

## 背景

ノードの見た目を制御するスタイリングシステムの設計方針を決める必要があった。

## 決定

CSS のセレクタ・カスケード・プロパティ名の仕組みを参考にした `.krs.style` ファイルを採用する。

## セレクタの詳細度

| セレクタ | スコア |
|---------|-------|
| 種別 | 1 |
| タグ / アノテーション | 10 |
| 種別+タグ | 11 |
| タグ+アノテーション | 20 |
| 種別+タグ+アノテーション | 21 |
| ID | 100 |

## 理由

- フロントエンド開発者にとって学習コストが低い
- カスケードの仕組みが直感的で予測しやすい
- プロパティ名を CSS 準拠にすることで既存の知識が活かせる

## 例外

`shape` プロパティのみ CSS に存在しないため karasu 固有のプロパティとして追加する。
（`box` / `user` / `cylinder` / `queue` / `hexagon` / `cloud` / `url("...")` を値として持つ）

## スタイルと意味の分離

タグ（`[external]` など）は**意味の宣言**であり、見た目の直接指定ではない。
見た目の制御は `.krs.style` のセレクタで行う。
これにより、同じ意味構造に対して異なるテーマを適用できる。
