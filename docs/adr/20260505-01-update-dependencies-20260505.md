# ADR-20260505-01: 依存パッケージ更新 — 2026-05-05

## Status

Accepted

## Context

This week's Dependabot batch contained a single PR:

| Package | From | To | Type | PR |
|---------|------|----|------|----|
| `pnpm/action-setup` | 6.0.1 | 6.0.3 | GitHub Actions | #1109 |

`.github/workflows/ci.yml` had previously been pinned to `pnpm/action-setup@v6.0.3` ahead of the Dependabot bump, so accepting the PR aligns the remaining workflow references with the pinned version.

### Initial concern: pnpm RC bundling

The release notes for `v6.0.3` state "Updated pnpm to v11.0.0-rc.5", which raised an initial concern about shipping a pnpm release candidate into CI builds. On further inspection of the release diff
([pnpm/action-setup@903f9c1](https://github.com/pnpm/action-setup/commit/903f9c1a6ebcba6cf41d87230be49611ac97822e))
this concern was resolved:

- `v6.0.2` already bundled the same pnpm RC line.
- The bundled pnpm is used only as a **bootstrap** binary inside the action.
- The pnpm version actually used by the project is resolved from the consumer's `package.json` `packageManager` field (or the action's `version` input).
- The `v6.0.2` fix (#230 — "pnpm self-update binary shadowed by bootstrap on PATH") is itself predicated on this bootstrap-vs-runtime separation.

Therefore the bundled RC has no effect on karasu's runtime pnpm and does not constitute a supply-chain risk for this upgrade.

### Policy implication for future Dependabot reviews

We considered tightening Dependabot policy (longer cooldown, prerelease ignore rules, process-doc additions) in response to this PR. After the bootstrap clarification above, no policy change is warranted: the existing 7-day cooldown plus per-PR release-note review is sufficient. Recording the reasoning here so the same investigation does not need to be repeated next time `pnpm/action-setup` ships an RC-bundled bootstrap.

## Decision

Adopt PR #1109. Keep current Dependabot configuration unchanged.

## Consequences

**Positive:**

- All `pnpm/action-setup` references across CI workflows are now consistent at `v6.0.3`.
- The `v6.0.2` bootstrap-shadowing fix (#230 upstream) is included, removing a latent failure mode for `pnpm self-update` inside the action.
- Future reviewers can consult this ADR instead of re-investigating the bundled-pnpm concern.

**Negative:**

- None identified.
