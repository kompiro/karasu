# ADR-20260410-01: Domain 間エッジと `[implicit]` 自動タグによる暗黙サービスエッジ

- **日付**: 2026-04-10
- **ステータス**: 決定済み
- **関連**: Issue #445

## 背景

`.krs` の `domain` ブロックはパーサーレベルで既にエッジ宣言 (`-> Contract` 等) を受け入れるが、view 抽出と描画段階でドメインレベルのエッジが処理されていなかった。ドメイン依存はサービス境界に依存しない本質的な関係であり、ドメインが別サービスに切り出されても依存の事実は変わらないため、一級市民として構文で表現すべきである。

加えて、クロスサービスのドメインエッジからサービス間エッジを派生させる際、明示エッジと区別した描画が必要となった。

## 決定

1. **パーサー変更なし** — ドメインブロックのエッジ宣言は既存実装で受け入れ済み
2. **ドメイン ID 一意性を error に格上げ** — `buildNodePathIndex()` のクロスサービス重複チェックを `warning` → `error` に変更
3. **サービスビューでのイントラサービスドメインエッジ描画** — view 抽出時に同一サービス内のドメイン間エッジを収集して描画する
4. **システムビューでの暗黙サービスエッジ派生** — クロスサービスのドメインエッジを検出し、`tags: ["implicit"]` を付与した `KrsEdge` を合成してビューに加える。既存の明示サービス間エッジと重複する場合は派生エッジを追加しない
5. **デフォルトスタイル** — `edge[implicit]` をアンバー `#F59E0B` の破線でスタイル解決する（ユーザーは `.krs.style` で上書き可能）
6. **ラベル表示** — 複数のドメインエッジが同一サービスペアに集約される場合は `"N domain edges"` 形式で件数表示し、クリックで各エッジの詳細を表示。単一エッジの場合はそのラベルをそのまま表示

## 理由

- **自動タグ + スタイル解決**: 既存のタグベーススタイル機構と一貫しており、`.krs.style` によるカスタマイズが可能。`[implicit]` は将来の他の派生エッジ（ghost エッジなど）にも再利用できる
- **アンバー `#F59E0B`**: ダーク背景での視認性が高く、既存のエッジ色と区別しやすい
- **ドメイン ID 一意性 error 化**: `-> Contract` の解決が曖昧にならないための前提条件。重複ドメインは ADR-20260411-02 の移行期例外のみ許容する
- **件数表示ラベル**: 複数のドメインエッジを 1 本の線で表現するとコンテキストが失われる。件数 + クリック詳細により責務の違いを保持する

## 却下した案

### 固定色ハードコード

`.krs.style` による上書きができず、カラーテーマ変更への対応が難しい。

### Ghost エッジ（`opacity: GHOST_OPACITY`）流用

既存の ghost 意味（「現在のビューの外にあるノードへの参照」）と混同される。視覚的に目立たず、暗黙依存が見落とされる可能性がある。意味的に不正確。

### 複数線描画（同一ペアを平行線で複数本）

ドメインエッジごとに線を描く案。エッジ交差ルーティングが複雑化するため現フェーズでは対象外。
