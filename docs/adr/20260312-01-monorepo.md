# ADR-20260312-01: モノレポ構成の採用

- **日付**: 2026-03-12
- **ステータス**: 決定済み

## 背景

karasu はパーサー・スタイル解決エンジン・プレビューUIという複数の関心事を持つ。
これらを単一パッケージにまとめるか、分割するかを検討した。

## 決定

npm workspaces を用いたモノレポ構成を採用する。

```
packages/
├── core/   ← パーサー・スタイル解決・SVGレンダラー（Pure TS）
└── app/    ← Vite + React のプレビューUI
```

## 理由

- `core` はブラウザ・Node.js どちらでも動く純粋な処理層として設計できる
- 将来的に `packages/cli` を追加するとき自然に対応できる
- `core` を独立した npm パッケージとして切り出すことも可能になる
- `app` の UI フレームワーク変更が `core` に影響しない

## 結果

- `core` は UI に依存しない純粋な処理層として設計する
- `app` は `core` を import して使う
- 技術スタック：TypeScript + Vite（app）+ Vitest（テスト）
