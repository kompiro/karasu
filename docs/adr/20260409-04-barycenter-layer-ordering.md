# ADR-20260409-04: 同レイヤー内コンテナ順序の Barycenter ヒューリスティックによる最適化

- **日付**: 2026-04-09
- **ステータス**: 決定済み
- **関連**: Issue #395, [ADR-20260408-02](20260408-02-deploy-layout-hierarchical-dag.md)

## 背景

ADR-20260408-02 で採用した階層 DAG レイアウトは、各レイヤー内のコンテナを挿入順に左から右へ並べていた。挿入順が悪いとレイヤー間のエッジが交差し、視認性が低下する。例えば `Layer 1: [A, B, C]`, `Layer 2: [X, Y, Z]` で `A→Z, B→Y, C→X` のとき、Layer 2 を `[Z, Y, X]` に並べ替えれば交差をゼロにできる。

## 決定

Barycenter ヒューリスティック（重心法）を採用し、各レイヤーのコンテナを「前レイヤーで隣接するコンテナの X 中心座標の平均」でソートする。前駆を持たないコンテナは `Infinity` を割り当てて末尾に置く（同値は挿入順を保持するため `sort` の安定性に依存）。一方向パス（layer 0 → last）のみで実装する。

```
barycenter(C) = mean(centerX[p] for p in predecessors(C) ∩ previousLayer)
```

実装後に Architecture diagram でも再利用できるよう、共通ユーティリティ `sortByBarycenter` を `layer-layout-logics.ts` に抽出する（ADR-20260411-01 参照）。

## 理由

- **実装が単純で実用品質が高い**: 1 パス、O(n log n)。Sugiyama フレームワークの標準ステップであり、多くのグラフ描画ツールで採用されている
- **直感的**: 「つながっている相手の近くに移動する」という動作が読み手の期待と一致する
- **deploy 図の規模に適合**: 各レイヤーのコンテナ数は実用上 2〜10 程度で、より高度なアルゴリズムの恩恵は限定的
- **将来拡張可能**: 品質向上が必要になれば双方向パス（案4）への拡張が容易

## 却下した案

### 案2: Median ヒューリスティック

中央値で順序を決める案。理論的には外れ値に頑健だが、deploy 図の規模では Barycenter との品質差は誤差レベル。同数中央値の扱い（左/右）が複雑になる。

### 案3: Sifting アルゴリズム

各コンテナを 1 つずつ最適位置に移動する案。O(n²) 以上のコストで実装も複雑。deploy 図の規模では過剰品質。

### 案4: 多方向反復パス（双方向 Barycenter）

上→下と下→上を交互に複数回適用する案。実装が複雑になる一方、deploy 図のレイヤー数は通常 3〜5 程度で多方向の効果が小さい。将来必要になった時点で拡張する。
