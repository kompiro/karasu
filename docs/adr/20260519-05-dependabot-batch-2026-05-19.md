# ADR-20260519-05: Dependabot Batch Triage (2026-05-19) — `pnpm/action-setup` 6.0.8 / `cloudflare/wrangler-action` 4.0.0

## 背景

2026-05-19 週の Dependabot version update バッチで起票された GitHub Actions の更新 PR 2 本を triage した。サプライチェーン攻撃のリスクを踏まえ、bump 種別を問わず両 PR を upstream まで遡ってリスク分析した。

| PR | 対象 | bump | 変更概要 | CI |
|----|------|------|----------|----|
| [#1426](https://github.com/kompiro/karasu/pull/1426) | `pnpm/action-setup` 6.0.5 → 6.0.8 | patch | pnpm を 11.1.1 へ更新、Windows での standalone + self-update 修正など fix のみ | 全 check green |
| [#1427](https://github.com/kompiro/karasu/pull/1427) | `cloudflare/wrangler-action` 3.15.0 → 4.0.0 | major | 唯一の major change は default Wrangler バージョンを v4（`latest`）へ更新 | 全 check green（deploy 系は skip）|

両リリースとも 2026-05-12 公開で、triage 時点（2026-05-19）で 7 日経過しており cooldown（全 semver レベル 7 日）を満たす。

### リスク分析の結論

- **両 PR**: PR がピンする commit SHA は upstream タグと一致（`#1426` の `0e279bb…` = `v6.0.8` 注釈タグの指す commit、`#1427` の `ebbaa158…` = `v4.0.0` タグ）。`v3.15.0...v4.0.0` / `v6.0.5...v6.0.8` の compare に新規 postinstall / prepare 等の lifecycle スクリプト追加なし。配布主体（`cloudflare` org / `pnpm` org）の変化なし、既知 advisory なし。
- **#1427 の major bump**: 実体は default wrangler バージョンが v4 になるのみ。karasu の `deploy.yml` / `preview.yml` は `wranglerVersion` を指定しておらず更新後は wrangler v4 が使われるが、利用コマンドは `pages deploy` のみで v4 でも継続サポートされている。

両 PR ともリスクレベル **low**。

## 決定

- **PR #1426 を squash merge する**（マージ済み）。`pnpm/action-setup` のパッチ更新で CI 専用、低リスク。
- **PR #1427 を squash merge する**。`cloudflare/wrangler-action` の major 更新だが、karasu の利用方法（`pages deploy`）に破壊的影響はなく低リスク。
- `.github/dependabot.yml` は変更しない。

## 理由

- いずれも CI / デプロイ専用の GitHub Action で、ランタイム成果物には影響しない。
- 両 PR とも全 required check が green で、secret-gated job（Playwright / ExTester / Cloudflare）は bot PR ポリシー通り skip。
- upstream まで遡った確認で配布主体・コード・lifecycle スクリプトに不審点がなく、cooldown も満たしている。
- #1427 は major だが、karasu は `wranglerVersion` を未指定で `pages deploy` のみ使うため、wrangler v4 への切り替えは破壊的影響を持たない。

## 却下した案

- **#1427 で `wranglerVersion: "3"` を明示ピンして v3 を維持する案**: 採用しなかった。wrangler v4 の `pages deploy` は安定しており、v3 に固定し続ける積極的な理由がない。必要が生じれば別 Issue で扱う。

## スコープ外

- `wranglerVersion` の明示ピン追加。
- 次回以降のバッチで複数 PR の競合や CI 失敗が発生する場合は、本 ADR と同様にバッチ ADR で triage する。
