# ADR-20260623-03: Dependabot Batch Triage (2026-06-23) — `pnpm/action-setup` 6.0.9

## 背景

2026-06-23 時点で開いていた Dependabot version update PR 1 本を triage した。サプライチェーン攻撃のリスクを踏まえ、bump 種別を問わず upstream まで遡ってリスク分析した。

| PR | 対象 | bump | 変更概要 | CI |
|----|------|------|----------|----|
| [#1703](https://github.com/kompiro/karasu/pull/1703) | `pnpm/action-setup` 6.0.8 → 6.0.9 | patch | バンドルする pnpm を 11.1.1 → 11.7.0 に更新するのみ | 全 check green（secret-gated job は skip）|

### リスク分析の結論

- **リリースノート / CHANGELOG**: v6.0.9 は `fix: update pnpm to v11.7.0`（[pnpm/action-setup#267](https://github.com/pnpm/action-setup/pull/267)）の 1 点のみ。
- **pin 整合性**: 本リポジトリは全 16 ワークフローで `pnpm/action-setup` を**フル commit SHA pin**（`# v6.0.8` コメント付き）。PR は SHA を `0e279bb…` → `0ebf471…` に更新する。annotated tag `v6.0.9` を dereference すると commit `0ebf471` に**完全一致**し、SHA とタグの不整合はない。
- **コード差分**: `0e279bb…0ebf471` の compare は `ahead_by: 1`（単一コミット）。変更は `dist/index.js`（バンドル再生成）と bootstrap lockfile 2 本（`exe-lock.json` / `pnpm-lock.json`）のみで、内容は bundled pnpm を `11.1.1 → 11.7.0` に bump するだけ。src ロジックの変更なし。不審なネットワーク送信・難読化の類はなし。
- **メンテナ・所有権の変化**: commit / tag いずれも **Zoltan Kochan**（`z@kochan.io`、pnpm メンテナ）が著者で GitHub **verified**。新規 publisher・リポジトリ移管・新メンテナ追加はなし。
- **lifecycle スクリプト**: GitHub Action のため postinstall / prepare 等なし。`@pnpm/exe` は元から `hasInstallScript:true`（挙動変化なし）。追加なし。
- **依存ツリー**: 新規 transitive 依存の追加なし。既存の `@pnpm/*` プラットフォームバイナリの version bump と `detect-libc ^2.0.3 → ^2.1.2` のみ。
- **公開からの経過時間**: 2026-06-15 公開（約 8 日）。本リポジトリの cooldown（全 semver レベル 7 日）を満たす。
- **既知 advisory**: 該当なし。

リスクレベル **low**。

## 決定

- **PR #1703 を squash merge する**（マージ済み）。`pnpm/action-setup` のパッチ更新で CI 専用、低リスク。
- `.github/dependabot.yml` は変更しない。

## 理由

- CI 専用の GitHub Action で、ランタイム成果物（core / app / cli / lsp / vscode）には影響しない。
- 全 required check が green で、secret-gated job（Playwright / ExTester / Cloudflare）は bot PR ポリシー通り skip。
- upstream まで遡った確認で、annotated tag が pin 先 SHA に一致し、配布主体（pnpm メンテナ）・コード差分・lifecycle スクリプトに不審点がない。変更内容はリリースノートと整合する。
- 全 16 ワークフローで SHA pin を維持しており、タグ改ざんによる差し替えリスクを受けない。
- 公開から 8 日経過し cooldown を満たす。

## 却下した案

- **マージを見送り様子見する案**: 採用しなかった。フル SHA pin される GitHub Action のパッチ更新で、annotated tag と pin 先 SHA が一致し差分もリリースノートと整合する本件では、待機の積極的理由がない。

## スコープ外

- `pnpm/action-setup` の major / minor 更新（本 triage 時点では未起票）。
- 次回以降のバッチで複数 PR の競合や CI 失敗が発生する場合は、本 ADR と同様にバッチ ADR で triage する。
