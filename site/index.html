What makes karasu different

Three-dimensional structure

Logical relationships between services and domains, the physical artifacts that get deployed, and the teams that own them — all in one .krs language. Conway's Law becomes part of the architecture conversation.

Scoped glance & drill-down

Progressive disclosure: limit how much is shown at once and descend into any node when detail is needed — instead of cramming a whole system into one bird's-eye diagram.

Human- and AI-co-authored

.krs stands on its own as a language humans read and write. That independence is what makes it bidirectional — AI can draft it, humans can hand-edit it, and back again.

See it in action

A few lines of .krs describe the model; karasu renders the diagram. 

system ECPlatform {
  label "EC Platform"

  user Customer [human] {
    label "Customer"
  }

  service ECommerce {
    label "EC Site"
  }
  service Payment [external] {
    label "Payment"
  }
  service Inventory [external] {
    label "Inventory"
  }

  Customer  -> ECommerce "Place an order"
  ECommerce -> Payment   "Process payment"
  ECommerce -> Inventory "Sync inventory"
}

Get started

In your browser

No install. A staged tutorial loads on first launch — edit, preview, drill down, and export SVG right away.

Open the app

On the command line

Render .krs files to SVG or serve a live preview locally.

npm install -g @kompiro/karasu-tools
karasu render index.krs -o arch.svg
karasu serve index.krs
